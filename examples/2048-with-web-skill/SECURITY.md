# Security Policy

## Supported versions

Security fixes are currently focused on the latest published release line.

| Version | Supported |
| --- | --- |
| `0.1.x` | Yes |
| Older versions | No |

## Reporting a vulnerability

Please do not open public GitHub issues for suspected security vulnerabilities.

Instead, report them privately to [dillion@dimbreak.com](mailto:dillion@dimbreak.com) with:

- a short summary of the issue
- affected versions or commit range if known
- reproduction steps or proof of concept
- expected impact
- any suggested mitigation if available

We will aim to:

- acknowledge receipt within 3 business days
- investigate and confirm severity
- coordinate a fix and release when needed

If the report turns out not to be a package-level vulnerability but an app-integration concern, we will still try to clarify the boundary and suggest the right mitigation.
